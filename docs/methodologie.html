Méthodologie

Cette page documente les méthodes d'analyse appliquées aux données publiques. L'objectif est la transparence totale : chaque score, chaque agrégat, chaque indicateur est reproductible à partir des données d'origine et des formules décrites ici.

Pipeline de traitement

  1. Collecte — Récupération via API ou CSV depuis les portails open data officiels
  2. Nettoyage — Suppression des doublons, traitement des valeurs manquantes et détection des valeurs aberrantes
  3. Normalisation — Réconciliation des codes communes INSEE via BAN avec 4 niveaux de fallback
  4. Agrégation — Calcul des indicateurs par commune : médiane, quartiles, distribution, évolution temporelle
  5. Croisement — Jointure des données via code INSEE pour analyses multisources

Réconciliation géographique

Pour chaque adresse ou localité, une réconciliation INSEE fiable est cruciale. Le pipeline suit 4 tentatives successives, chacune avec un seuil de confiance minimum :

Tentative Méthode Seuil de confiance
1 Match exact BAN API ≥ 0.95 (très élevé)
2 Géocodage inverse (coordonnées GPS) ≥ 0.90 (bon)
3 Match partiel BAN API ≥ 0.80 (acceptable)
4 Résolution code postal seul ≥ 0.50 (faible)

Règle critique : Un score de confiance inférieur à 0.80 déclenche un avertissement visible et rend l'enrichissement approximatif.

Indicateurs calculés

Pour chaque commune, les analyses produisent les indicateurs suivants :

Marché immobilier (DVF)

Prix au m² (médiane, Q1, Q3), évolution sur 12 et 24 mois, volume de transactions, délai médian de vente

Énergie (DPE)

Répartition des classes A-G, % passoires (F+G), consommation moyenne, GES moyen, score énergie

Équipements (BPE)

Score de proximité (0-100) : transports, commerces, santé, éducation. Walkability index

Démographie (INSEE)

Population, composition socioprofessionnelle, âge moyen, densité, évolution annuelle

Scores de confiance

Chaque indicateur produit un score de confiance (0-100) basé sur :

  • Volume de données — Nombre de transactions ou diagnostics (40 points max)
  • Fraîcheur — Ancienneté des données les plus récentes (30 points max)
  • Croisement de sources — Nombre de sources différentes convergentes (30 points max)

Un score < 50 indique des résultats très approximatifs. Un score ≥ 75 indique une confiance raisonnable.

Limitations et avertissements

  • DVF — Pas de données de transactions de gré à gré, SCI, donations. Délai de publication : ~6 mois.
  • DPE — Diagnostic depuis juillet 2021 seulement. Couverture inégale par région.
  • BPE — Mise à jour annuelle. Couverture variable. Peut être incomplet en zones rurales.
  • Communes < 5 transactions/an — Résultats approximatifs, signalés avec avertissement.
  • Données anciennes — Si données > 6 mois : avertissement de fraîcheur.

Reproductibilité

Pour chaque commune, Intent Analytics documente :

  • Les sources utilisées et leur date de récupération
  • Le nombre de points de données (transactions, diagnostics, etc.)
  • Les formules exactes des agrégats (médiane, quartiles, évolution)
  • Le score de confiance et ses composantes
  • Les avertissements applicables

Tout résultat peut être reproduit par un tiers en appliquant cette documentation.
