Méthodologie

Cette page documente les méthodes d'analyse appliquées aux données publiques. L'objectif est la transparence totale : chaque score, chaque agrégat, chaque indicateur est reproductible à partir des données d'origine et des formules décrites ici.

Pipeline de traitement

  1. Collecte — Récupération via API ou CSV depuis les portails open data officiels
  2. Nettoyage — Suppression des doublons, traitement des valeurs manquantes et détection des valeurs aberrantes
  3. Normalisation — Réconciliation des codes communes INSEE via BAN avec 4 niveaux de fallback
  4. Agrégation — Calcul des indicateurs par commune : médiane, quartiles, distribution, évolution temporelle
  5. Croisement — Jointure des données via code INSEE pour analyses multisources

Réconciliation géographique

Pour chaque adresse ou localité, une réconciliation INSEE fiable est cruciale. Le pipeline suit 4 tentatives successives, chacune avec un seuil de confiance minimum :

Tentative Méthode Seuil de confiance
1 Match exact BAN API ≥ 0.95 (très élevé)
2 Géocodage inverse (coordonnées GPS) ≥ 0.90 (bon)
3 Match partiel BAN API ≥ 0.80 (acceptable)
4 Résolution code postal seul ≥ 0.50 (faible)

Règle critique : Un score de confiance inférieur à 0.80 déclenche un avertissement visible et rend l'enrichissement approximatif.

Indicateurs calculés

DVF — Prix immobiliers

Prix médian au m², quartiles Q1/Q3, volume de ventes annuel, évolution 12m/24m, délai moyen de vente. Minimum 3 transactions pour inclusion.

DPE — Performance énergétique

Répartition classes A-G en %, consommation kWh/m²/an, GES kg CO2/m²/an, score composite 0-100, pourcentage passoires F+G.

BPE — Équipements locaux

Score par catégorie (transports, éducation, santé, commerce), score global 0-100 reflétant l'accessibilité locale.

IRIS — Démographie

Population, densité, pyramide des âges, taux d'activité, revenus, condition de logement agrégés au niveau communal.

Indice de confiance

Chaque résultat s'accompagne d'un indice de confiance 0-100. Cet indice reflète la fiabilité statistique estimée en fonction du volume de données, de la fraîcheur des données et du nombre de sources croisées.

Plage de score Interprétation Recommandation
80-100 Fiabilité très élevée Utiliser directement pour analyses décisionnelles
60-79 Fiabilité correcte Utiliser avec confiance, vérifier contexte local
40-59 Fiabilité limitée Interpréter comme indicateur exploratoire, non décisionnel
0-39 Fiabilité faible Données insuffisantes, données approximatives uniquement

Sources et traçabilité

Chaque donnée affichée mentionne sa source originelle. Les transformations appliquées sont documentées dans les logs de collecte. Une donnée sans source identifiée n'est jamais affichée.

L'audit complet des imports, agrégations et calculs est conservé dans les tables data_import_logs et audit_log, garantissant la reproductibilité.

Limitations statistiques

Les analyses produites reposent sur des données publiques qui peuvent présenter des limites intrinsèques :

  • Complétude : DVF exclut SCI et donations. DPE couverture depuis 2021 seulement. BPE couverture inégale.
  • Fraîcheur : DVF délai 6 mois. DPE mise à jour continue mais décalage 1-2 mois possibles. BPE annuelle.
  • Géocodage : Certaines adresses restent non localisées malgré les 4 niveaux de réconciliation. Un score < 0.50 signale l'impasse.
  • Valeurs aberrantes : Outliers conservés sauf si > 5 sigma. Mentionnés en warning quand ils biaisent la médiane.
