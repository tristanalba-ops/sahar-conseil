01 · Genèse

Un socle cohérent pour lire le territoire.

Intent Analytics est né d'un constat simple : les données publiques françaises sont abondantes, mais dispersées. Elles deviennent utiles quand on les croise.

Chaque ministère, chaque opérateur d'État publie ses données. La DGFiP documente chaque vente immobilière. L'ADEME collecte chaque diagnostic énergétique. L'INSEE cartographie les équipements et la démographie. L'IGN maintient la base d'adresses. Le CSTB décrit le bâti. L'ANCT géocode.

Pris isolément, chacun de ces référentiels apporte une lecture partielle. Pris ensemble, ils racontent le territoire.

Intent Analytics collecte, nettoie, géocode et croise ces sept bases en un référentiel unique, rafraîchi en continu. Le projet expose des explorations statistiques documentées, un indice de confiance par enregistrement, et une trace d'audit complète.

Trois convictions

La fiabilité avant le volume.

Un enregistrement douteux est pire qu'un enregistrement manquant. Chaque donnée porte un score de qualité vérifiable. Les matches faibles sont flagués puis purgés automatiquement.

La donnée brute est ouverte.

Toutes les sources utilisées sont déjà publiques, sous licence ouverte. Intent Analytics ne crée pas de donnée — il en facilite la lecture.

La méthodologie est publique.

Les règles de croisement, les seuils de confiance, les filtres qualité : tout est documenté. Reproductibilité par défaut.

Ce que le projet n'est pas

  • Ce n'est pas un outil d'estimation automatique : les explorations publiées ne constituent pas une expertise.
  • Ce n'est pas un produit commercial : l'objectif est la lecture et la compréhension, pas la transaction.
  • Ce n'est pas un référentiel officiel : les données sources restent celles des producteurs publics.
