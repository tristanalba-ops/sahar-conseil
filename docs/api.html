Documentation API

L'API Intent Analytics vous permet d'intégrer des estimations immobilières basées sur les données publiques françaises (DVF, DPE ADEME, INSEE) directement dans vos applications.

Accès rapide Vous avez besoin d'une clé API ? Voir nos tarifs et obtenez votre clé en quelques minutes.

Authentification

Toutes les requêtes doivent inclure votre clé API dans l'en-tête HTTP x-api-key.

x-api-key: ia_live_xxxxxxxxxxxxxxxxxxxx
Ne partagez jamais votre clé API dans du code côté client ou dans un dépôt public. Utilisez des variables d'environnement (process.env.IA_API_KEY).

URL de base

Toutes les requêtes sont envoyées à l'URL suivante :

https://wwvdpixzfaviaapixarb.supabase.co/functions/v1

POST /estimate

Retourne une estimation du prix d'un bien immobilier en se basant sur les transactions DVF récentes et les données contextuelles (DPE, INSEE).

POST https://wwvdpixzfaviaapixarb.supabase.co/functions/v1/estimate

Corps de la requête

Le corps de la requête doit être du JSON (Content-Type: application/json).

Champ Type Requis Description
city string Requis Nom de la commune (ex : "Paris 15", "Lyon 3", "Bordeaux")
property_type string Requis Type de bien : "apartment" ou "house"
surface_m2 number Requis Surface habitable en m² (entre 5 et 2000)
rooms integer Optionnel Nombre de pièces principales (1–20). Améliore la précision.
year_built integer Optionnel Année de construction (ex : 1985). Utilisé pour l'abattement vétusté.
floor integer Optionnel Étage du bien (0 = rez-de-chaussée). Appartements uniquement.
has_parking boolean Optionnel Indique si une place de parking est incluse.

Réponse

En cas de succès, l'API retourne un objet JSON avec les champs suivants :

Champ Type Description
estimated_price number Prix estimé en euros (valeur centrale)
price_low number Borne basse de l'intervalle de confiance (€)
price_high number Borne haute de l'intervalle de confiance (€)
price_per_m2 number Prix estimé au m² (€/m²)
confidence_score integer Score de confiance de 0 à 100 (basé sur la densité et la fraîcheur des données DVF)
comparable_count integer Nombre de transactions DVF comparables utilisées
data_freshness string Date de la transaction DVF la plus récente prise en compte (ISO 8601)
sources array<string> Sources de données utilisées (ex : ["DVF", "DPE_ADEME", "INSEE"])
request_id string Identifiant unique de la requête (UUID v4) — à conserver pour le support

Exemples de code

Requête — appartement 3 pièces à Lyon

curl -X POST \
  "https://wwvdpixzfaviaapixarb.supabase.co/functions/v1/estimate" \
  -H "Content-Type: application/json" \
  -H "x-api-key: ia_live_xxxxxxxxxxxxxxxxxxxx" \
  -d '{
    "city": "Lyon 3",
    "property_type": "apartment",
    "surface_m2": 65,
    "rooms": 3,
    "year_built": 1978,
    "floor": 4
  }'
const response = await fetch(
  "https://wwvdpixzfaviaapixarb.supabase.co/functions/v1/estimate",
  {
    method: "POST",
    headers: {
      "Content-Type": "application/json",
      "x-api-key": process.env.IA_API_KEY
    },
    body: JSON.stringify({
      city:          "Lyon 3",
      property_type: "apartment",
      surface_m2:    65,
      rooms:         3,
      year_built:    1978,
      floor:         4
    })
  }
);

const data = await response.json();
console.log(data.estimated_price); // ex: 287500
import requests
import os

url = "https://wwvdpixzfaviaapixarb.supabase.co/functions/v1/estimate"

headers = {
    "Content-Type": "application/json",
    "x-api-key":    os.getenv("IA_API_KEY")
}

payload = {
    "city":          "Lyon 3",
    "property_type": "apartment",
    "surface_m2":    65,
    "rooms":         3,
    "year_built":    1978,
    "floor":         4
}

resp = requests.post(url, json=payload, headers=headers)
resp.raise_for_status()

data = resp.json()
print(data["estimated_price"])  # ex: 287500

Codes d'erreur

L'API utilise les codes de statut HTTP standard. Le corps de toute réponse d'erreur inclut un objet { "error": "...", "request_id": "..." }.

400 Bad Request Un champ requis est manquant ou invalide. Vérifiez city, property_type et surface_m2.
401 Unauthorized Clé API absente ou invalide. Vérifiez l'en-tête x-api-key.
403 Forbidden La clé API est valide mais n'a pas les droits pour cette opération (plan insuffisant).
404 Not Found La commune spécifiée dans city est introuvable dans notre référentiel.
422 Unprocessable Entity Les données DVF sont insuffisantes pour cette zone (comparable_count < 3). Élargissez la commune ou acceptez un confidence_score bas.
429 Too Many Requests Quota journalier atteint. L'en-tête Retry-After indique les secondes avant remise à zéro (minuit UTC).
500 Internal Server Error Erreur serveur. Réessayez avec un backoff exponentiel. Contactez le support avec le request_id si l'erreur persiste.
503 Service Unavailable Maintenance planifiée ou surcharge temporaire. Consultez la page de statut.

Rate limits
